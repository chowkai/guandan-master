## 附录

### Bug 提交流程

1. 发现 Bug
2. 记录复现步骤
3. 确定优先级
4. 提交到 Bug 追踪清单
5. 分配修复人
6. 修复并验证
7. 更新状态

### Bug 优先级判定标准

**P0 - 致命**:
- 应用崩溃
- 数据丢失
- 无法进行游戏

**P1 - 严重**:
- 核心功能不可用
- 游戏规则错误
- 严重影响体验

**P2 - 一般**:
- 功能缺陷
- 性能问题
- 影响部分用户

**P3 - 轻微**:
- UI 问题
- 文案错误
- 建议性改进
