## 📦 传令兵系统

### 核心设计

**命令传递机制**：
- ✅ 玩家不能直接指挥个体，只能对编制下达命令
- ✅ 命令通过传令兵传递到目标编制
- ✅ 传令兵到达后，命令才生效
- ✅ 小范围内（视距/声距）可快速命令（旗鼓号）

### 传令兵自主机制

**玩家可调整的参数**：
| 参数 | 说明 | 影响 |
|------|------|------|
| **命令紧急度** | 高/中/低 | 决定传令兵优先级 |
| **传递方式** | 自动多派分路 | 提高成功率 |

**AI 自动执行**：
```
玩家决策:
  "第 X 师前往 A 地区驻防"（紧急度：高）

AI 自动执行:
  1. 自动派遣多个传令兵
  2. 自动分时分路传递
  3. 自动规避危险区域
  
意外事件:
  - 传令兵可能被拦截
  - 传令兵可能迷路（无识途技能）
  - 传令兵可能延误（天气/地形）
```

**传令兵技能**：
| 技能 | 效果 |
|------|------|
| **识途** | 远距离定位目标，不会迷路 |
| **护身** | 减免恐慌度，提高生存率 |
| **快速** | 提高移动速度 |

## 👥 个体全生命周期系统

### 个体行为三原则

**第一原则：确保生存**
```
生存威胁因素:
- 饥渴：饮食耗尽
- 伤重：生命值过低
- 敌人袭击：遭遇敌方部队
- 力竭：体力耗尽

应对行为:
- 补给快用完 + 离补给点太远 → 惊慌度上升 → 士气下降
- 达到临界点 → 主动离开部队寻找补给或投降
- 落单遭遇敌方 → 逃跑或投降

特殊技能豁免:
- 斥候/传令兵：特殊技能减免恐慌效果
```

**第二原则：依附编制**
```
编制的作用:
1. 补给来源：只有编制内才能获得稳定补给
2. 集群威慑：借助集体力量对抗敌人
3. 命令传达：接收上级指令，协同行动
4. 归属感：降低恐慌度，提高士气

溃散机制:
- 个体逃离编制超过视距/声距 → 失去方向感
- 没有上帝视角 → 随机选择方向乱跑
- 斥候/传令兵有识途技能 → 可远距离定位

溃散结局:
- 遇上敌方部队 → 俘虏
- 遇上己方补给点 → 收容为散兵等待再分配
- 补给耗尽/体力耗尽 → 死亡
- 一段时间后 → 失踪（从历史中消失）
```

**第三原则：渴望晋升**
```
晋升机制:
- 系统根据能力与军功自动提拔
- 玩家也可以干预（满足趣味）
- 个体会努力完成上级交派的任务

军功获取:
- 击毙敌人、击伤敌人
- 完成任务、立功表现
- 逃跑/投降会扣分
```

## 💥 群体崩溃机制

### 凝聚力系统

**群体性溃散**：
```
凝聚力 = f(士气，军官存在，伤亡比例，补给状态)

崩溃流程:
1. 凝聚力下降到阈值 → 个体开始逃跑
2. 逃跑传染 → 群体性崩溃
3. 军官阵亡 → 加速崩溃
4. 完全崩溃 → 编制解散，个体溃散

影响因素:
- 军官存在：提高凝聚力阈值
- 老兵比例：提高凝聚力
- 补给充足：提高凝聚力
- 连胜战绩：提高凝聚力
```

### 数据可视化

**显示策略**：
| 数据类型 | 是否显示 | 说明 |
|---------|---------|------|
| **士气** | ✅ 显示 | 表面数据 |
| **补给** | ✅ 显示 | 表面数据 |
| **凝聚力** | ✅ 显示 | 表面数据（简化） |
| **惊慌度** | ❌ 隐藏 | 深层数据 |
| **个体属性** | ❌ 隐藏 | 深层数据 |
| **恐慌阈值** | ❌ 隐藏 | 深层数据 |

**设计原则**：只显示表面数据，避免信息过载

## 🛠️ 装备系统细节

### 装备参数体系

**投掷物**（子弹、弓箭等）：
| 参数 | 说明 | 影响 |
|------|------|------|
| **射程** | 有效攻击距离 | 决定攻击范围 |
| **重量** | 投掷物质量 | 影响负重、动能 |
| **破甲度** | 穿透护甲能力 | 决定对护甲伤害 |
| **爆炸范围** | 爆炸影响半径 | 决定范围伤害 |

**非投掷近战武器**：
| 参数 | 说明 | 影响 |
|------|------|------|
| **尖锐度** | 武器锋利程度 | 决定对护甲伤害 |
| **重量** | 武器质量 | 影响挥舞速度、负重 |
| **长度** | 武器攻击距离 | 决定攻击范围 |

**伤害计算**：
```
伤害能量 = f(重量，速度)
实际伤害 = f(伤害能量，命中率，护甲度，尖锐度/破甲度)

结果:
- 轻伤：属性暂时下降
- 重伤：无法战斗，需要医疗
- 死亡：从游戏中移除
```

### 装备 DIY 编辑器

**核心设计理念**：
- ✅ 系统预设常见弹药/装备
- ✅ 玩家可在编辑器中 DIY 任何装备
- ✅ 包括原子弹等超级武器
- ✅ 基础数据可自由组合

**可编辑参数**：
```
装备类型:
- 武器类：投掷/非投掷
- 防具类：衣服/护甲/头盔
- 工具类：通讯设备/工程工具
- 载具类：车辆/马匹/船只

基础数据组合:
- 射程、重量、破甲度、爆炸范围
- 尖锐度、长度、速度
- 耐久度、折旧率
- 技能要求、训练时间
```

### 装备折旧与维修

**折旧机制**：
- ✅ 装备有折旧度
- ✅ 可以修复，不需要玩家操心动手
- ✅ 玩家只要给部队配工匠或建维修厂
- ✅ 缴获装备可用（需士兵有技能/运输能力）

## 🏛️ 经济系统

### 核心设计

**抽象层**：
- ✅ 只有"资金"概念
- ✅ 所有物资以物质形式存在（重量、体积）
- ✅ 需要运输、存储

**地域属性**：
| 属性 | 说明 | 影响 |
|------|------|------|
| **人口** | 地区人口数量 | 决定征兵/招工进度 |
| **经济程度** | 地区经济发展水平 | 决定税收收入 |

**税收系统**：
```
税收 = f(占领区域，人口，经济程度)

用途:
- 训练士兵（消耗资金 + 时间）
- 购买物资（从仓库或市场）
- 维护装备（维修/折旧）
```

### 不想太复杂的系统

**简化设计**：
- ❌ 无外交系统（增加复杂度）
- ❌ 经济不想太复杂（只有资金 + 物资）
- ❌ 不攀科技树（由编辑器决定）

## 🏗️ 建筑系统

### 可移动经济建筑

**设计**：
- ✅ 经济建筑也可移动
- ✅ 需要付出迁移代价
- ✅ 目的地需要能够搭建

**限制**：
```
特殊建筑限制:
- 矿场：只能在矿脉上
- 农场：只能在耕地上
- 工厂：需要基础设施
- 港口：只能在水边
```

### 建筑占领与平民

**建筑内人员**：
- ✅ 经济建筑内有平民
- ✅ 大部分是被俘敌方人员/工人
- ✅ 人的个性随机生成

**平民行为**：
```
平民类型:
- 大部分：被俘人员，正常工作
- 少部分忠诚者：可能逃跑或反抗
- 有从军经历者：可能组织抵抗

建筑占领:
- 建筑可被破坏
- 建筑可被占领
- 占领后平民可能被俘或逃跑
```

## 🎯 核心三问答案

### A. 乐趣点

**战略布局 + 人性洞察 + 历史沉浸感**
- "运筹帷幄之中，决胜千里之外"的成就感
- "见证历史，改写历史"的沉浸感
- 不是微操快感（不能直接指挥）
- 不是数值碾压（不能违背人性）

### B. 指挥边界

**玩家可指挥任意编制**
- 包括最底层的班或经济建筑
- 不能直接指挥个体
- 命令通过传令兵传递
- 经济建筑也可移动（需付出代价）

### C. 时代范围

**由地图/战役编辑器决定**
- 不攀科技树
- 装备/通讯/马匹从经济建筑产出
- 士兵需要训练时间和资金
- 玩家复现历史上所有战争
