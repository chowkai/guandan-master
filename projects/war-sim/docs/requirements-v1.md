## 📋 核心理念

**战争是由无数微小个体的决策构成的**。每个士兵不是冷冰冰的数字，而是有生存本能、有归属感、有晋升欲望的"人"。

**核心思想**：
- ✅ 每个单位都是独立个体，有自己的生命周期
- ✅ 个体行为遵循优先级原则（生存 > 编制 > 晋升）
- ✅ 玩家不能违背"人性"（不能随意牺牲个体）
- ✅ 宏观战争结局由微观个体决策汇聚而成

## 🧠 个体行为优先级

### 第一原则：确保生存

**生存威胁因素**：
| 威胁 | 描述 | 应对行为 |
|------|------|---------|
| **饥渴** | 饮食耗尽 | 寻找补给/投降/逃跑 |
| **伤重** | 生命值过低 | 寻求医疗/撤退 |
| **敌人袭击** | 遭遇敌方部队 | 战斗/逃跑/投降 |
| **力竭** | 体力耗尽 | 休息/求救 |

**恐慌机制**：
```
恐慌度 = f(饮食剩余，距离补给点，敌方数量，己方数量，士气)

恐慌度阈值：
- 30%: 开始焦虑，效率下降
- 60%: 士气快速下降
- 80%: 考虑逃跑/投降
- 100%: 强制溃散（特殊技能可减免）
```

**特殊技能豁免**：
| 技能 | 效果 | 适用兵种 |
|------|------|---------|
| **识途** | 远距离定位目标，不会迷路 | 斥候、传令兵 |
| **护身** | 减免恐慌度积累 | 斥候、传令兵 |
| **坚韧** | 提高恐慌阈值 | 老兵、军官 |

---

### 第二原则：依附编制

**编制的作用**：
1. **补给来源**：只有编制内才能获得稳定补给
2. **集群威慑**：借助集体力量对抗敌人
3. **命令传达**：接收上级指令，协同行动
4. **归属感**：降低恐慌度，提高士气

**溃散机制**：
```
当个体逃离编制超过视距/声距范围：
1. 失去补给来源 → 饮食消耗加快
2. 失去集群保护 → 遭遇敌人必败
3. 失去方向感 → 随机移动（无识途技能）
4. 士气持续下降 → 最终崩溃

可能的结局：
- 遇上敌方 → 俘虏/阵亡
- 遇上己方补给点 → 收容为散兵
- 体力耗尽 → 死亡
- 一段时间后 → 失踪（从历史中消失）
```

---

### 第三原则：渴望晋升

**晋升机制**：
- 系统默认**100% 自动晋升**（按军功/能力/资历）
- 玩家可干预（满足"当长官"趣味），但有士气代价
- 填补编制内空缺岗位

**军功获取**：
| 行为 | 军功值 |
|------|--------|
| 击毙敌人 | +10 |
| 击伤敌人 | +5 |
| 完成任务 | +20 |
| 立功表现 | +15 |
| 逃跑/投降 | -50 |
| 被俘 | -30 |

## 🏗️ 命令系统

### 核心原则

**玩家不能直接指挥个体**：
- ❌ 玩家**不能**对任何个体直接下命令
- ✅ 玩家只能对**某一层级编制**下达命令
- ✅ 命令需要**传令兵送达**后才生效
- ✅ 传令延迟取决于距离和通讯手段

### 命令传达流程

```
玩家决策 → 总部下达命令 → 传令兵出发 → 到达下级总部 → 命令生效 → 执行
```

### 通讯手段演进

| 时代 | 通讯方式 | 范围 | 延迟 |
|------|---------|------|------|
| **古代** | 传令兵（骑马） | 无限 | 高（距离/速度） |
| **古代** | 旗鼓号 | 视距/声距 | 低（信息简单） |
| **近代** | 传令兵（车辆） | 无限 | 中 |
| **现代** | 电台/电话 | 无限 | 即时 |
| **现代** | 数据链 | 无限 | 即时 + 可视化 |

### 传令兵机制

**自动多路径传令**：
- 玩家可调整命令**紧急度**
- 紧急度高 → 自动多派传令兵、分路传送
- 斥候机制相同（编制组建时多拨人手/资源）

**意外情况**：
- 传令兵可能被拦截 → 命令失效
- 多路径传令提高成功率
- 这是战略层面考虑，**非玩家微操**

## 🧱 群体崩溃机制

### 核心原则

**溃散是群体性的，不是个体行为**：
- ✅ 团队凝聚力决定崩溃阈值
- ✅ 崩溃一旦开始，会**传染扩散**
- ✅ 如多米诺骨牌效应

### 崩溃机制

```
个体恐慌 → 群体恐慌 → 凝聚力下降 → 群体崩溃 → 传染扩散
```

**凝聚力影响因素**：
| 因素 | 影响 | 说明 |
|------|------|------|
| **军官能力** | 高 | 优秀军官稳定军心 |
| **老兵比例** | 高 | 老兵越多越稳定 |
| **战况** | 高 | 优势时凝聚力强 |
| **补给状况** | 中 | 充足补给提升士气 |
| **编制规模** | 中 | 大编制更稳定 |
| **民族/语言** | 中 | 同质化程度高更团结 |

**崩溃传播**：
```
传播速度 = f(恐慌度，凝聚力，地形，军官)

- 恐慌度高 → 传播快
- 凝聚力低 → 传播快
- 开阔地形 → 传播快（视野好，易看到别人逃跑）
- 军官阵亡 → 传播加速

传播范围:
- 同编制内：快速传播
- 邻近编制：慢速传播（视觉/听觉影响）
- 远处编制：不传播
```

**阻止崩溃**：
| 方法 | 效果 | 条件 |
|------|------|------|
| **军官督战** | 高 | 军官存活且在场 |
| **预备队投入** | 高 | 有预备队可用 |
| **撤退命令** | 中 | 命令能传达到 |
| **胜利消息** | 中 | 友军获胜消息 |

## 📊 技术可行性

### 核心挑战

**问题**：成千上万个个体，每个都需要实时做出决策，性能是否可行？

### 推荐方案：LOD + 时间分片 + ECS

#### 1. LOD 决策系统（Level of Detail）

```
根据距离和重要性，采用不同更新频率：

LOD0 (重要单位 - 玩家附近/关键单位):
  - 每帧更新 (60Hz)
  - 完整 AI 决策
  
LOD1 (普通单位 - 可见范围):
  - 每秒 10 次 (10Hz)
  - 简化 AI 决策
  
LOD2 (远处单位 - 不可见但附近):
  - 每秒 1 次 (1Hz)
  - 极简决策
  
LOD3 (超远单位 - 战略层面):
  - 每分钟 1 次 (1/60Hz)
  - 统计模拟
```

#### 2. 时间分片（Time Slicing）

```python
# 每帧只处理部分单位
units_per_frame = 1000  # 每帧处理 1000 个单位
```

#### 3. ECS 架构（Entity Component System）

```
优势：
- 数据与逻辑分离
- 缓存友好
- 易于并行处理
- 支持海量单位
```

### 性能目标

| 规模 | 单位数 | 目标 FPS | 更新策略 |
|------|--------|---------|---------|
| **小队战斗** | 100 | 60 | 全量更新 |
| **战役级** | 10,000 | 60 | LOD + 分片 |
| **战争级** | 100,000 | 30+ | LOD3 统计模拟 |

## 🎮 游戏设计

### 玩家价值体现

既然玩家不能直接指挥个体，那玩家的价值体现在：

| 作用 | 说明 |
|------|------|
| **战略规划** | 制定整体作战计划 |
| **资源调配** | 分配补给/装备/援军 |
| **人事任命** | 选择编制指挥官（可选干预） |
| **时机把握** | 决定进攻/撤退时机 |
| **编制调整** | 组建/整编/拆分编制 |

### 数据可视化原则

**只给表面数据，隐藏深层机制**：

| 数据类型 | 是否显示 | 说明 |
|---------|---------|------|
| **生命值** | ✅ 显示 | 表面数据 |
| **士气** | ✅ 显示 | 表面数据 |
| **恐慌度** | ❌ 隐藏 | 深层机制 |
| **凝聚力** | ❌ 隐藏 | 深层机制 |
| **恐慌阈值** | ❌ 隐藏 | 深层机制 |

### 时间流速控制

| 模式 | 效果 |
|------|------|
| **暂停** | 所有单位停止思考，玩家可调整部署 |
| **正常** | 完整模拟（60Hz） |
| **加速** | LOD 自动降级（牺牲精度换性能） |
