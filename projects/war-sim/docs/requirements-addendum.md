## 📋 用户确认的核心机制

### 1️⃣ 命令系统 - 玩家不能直接指挥个体

**核心原则**：
- ❌ 玩家**不能**对任何个体直接下命令
- ✅ 玩家只能对**某一层级编制**下达命令
- ✅ 命令需要**传令兵送达**后才生效
- ✅ 传令延迟取决于距离和通讯手段

**命令传达流程**：
```
玩家决策
   ↓
总部下达命令
   ↓
传令兵出发（骑马/电台/旗鼓号）
   ↓
到达下级编制总部
   ↓
命令生效
   ↓
下级编制执行（个体跟随编制行动）
```

**通讯手段演进**：
| 时代 | 通讯方式 | 范围 | 延迟 |
|------|---------|------|------|
| **古代** | 传令兵（骑马） | 无限 | 高（距离/速度） |
| **古代** | 旗鼓号 | 视距/声距 | 低（但信息简单） |
| **近代** | 传令兵（车辆） | 无限 | 中 |
| **现代** | 电台/电话 | 无限 | 即时 |
| **现代** | 数据链 | 无限 | 即时 + 可视化 |

**游戏设计意义**：
- ✅ 玩家不能"微操"每个士兵
- ✅ 命令传达有延迟，需要预判
- ✅ 传令兵被拦截 = 命令失效
- ✅ 符合真实战争逻辑

### 2️⃣ 溃散机制 - 群体性崩溃

**核心原则**：
- ✅ 溃散是**群体性**的，不是个体行为
- ✅ 团队凝聚力决定崩溃阈值
- ✅ 崩溃一旦开始，会**传染扩散**

**崩溃机制**：
```
个体恐慌度 → 影响周围单位 → 群体恐慌度 → 凝聚力下降
                                                      ↓
                                         凝聚力 < 阈值 → 群体崩溃
                                                      ↓
                                         溃散开始传播（如多米诺骨牌）
```

**凝聚力影响因素**：
| 因素 | 影响 | 说明 |
|------|------|------|
| **军官能力** | 高 | 优秀军官稳定军心 |
| **老兵比例** | 高 | 老兵越多越稳定 |
| **战况** | 高 | 优势时凝聚力强 |
| **补给状况** | 中 | 充足补给提升士气 |
| **编制规模** | 中 | 大编制更稳定 |
| **民族/语言** | 中 | 同质化程度高更团结 |

**崩溃传播**：
```
传播速度 = f(恐慌度，凝聚力，地形，军官)

- 恐慌度高 → 传播快
- 凝聚力低 → 传播快
- 开阔地形 → 传播快（视野好，容易看到别人逃跑）
- 军官阵亡 → 传播加速

传播范围:
- 同编制内：快速传播
- 邻近编制：慢速传播（视觉/听觉影响）
- 远处编制：不传播
```

**阻止崩溃**：
| 方法 | 效果 | 条件 |
|------|------|------|
| **军官督战** | 高 | 军官存活且在场 |
| **预备队投入** | 高 | 有预备队可用 |
| **撤退命令** | 中 | 命令能传达到 |
| **胜利消息** | 中 | 友军获胜消息 |

### 3️⃣ 晋升系统 - 默认全自动

**核心原则**：
- ✅ 系统默认**100% 自动晋升**
- ✅ 玩家干预是**可选的**（满足某种趣味）
- ✅ 玩家干预需要付出代价（士气/效率）

**自动晋升逻辑**：
```
当编制内出现岗位空缺：
  1. 系统筛选符合条件的人选
  2. 按优先级排序（军功 > 能力 > 资历）
  3. 自动任命
  4. 通知玩家（可选）
```

**玩家干预方式**：
| 干预类型 | 说明 | 代价 |
|---------|------|------|
| **手动任命** | 玩家指定人选 | 士气 -5（任人唯亲） |
| **破格提拔** | 跳过正常流程 | 士气 -3（不公平感） |
| **越级指挥** | 直接指挥下级 | 士气 -10（指挥链混乱） |
| **撤职** | 免除职务 | 士气 -15（除非有正当理由） |

**设计意义**：
- ✅ 默认全自动，减少玩家操作负担
- ✅ 保留干预选项，满足"当长官"的趣味
- ✅ 干预有代价，防止滥用

### 4️⃣ 履历系统 - UI 设计问题

**核心原则**：
- ✅ 这是 UI 设计问题，不是系统问题
- ✅ 需要方便玩家查看单位履历
- ✅ 但不要过度干扰游戏流程

**UI 设计方案**：
```
方案 A: 悬停预览
- 鼠标悬停在单位/编制上
- 显示简要信息（名称、状态、军功）
- 点击展开详细履历

方案 B: 编制面板
- 打开编制管理面板
- 树形结构显示层级
- 点击任意节点查看详细信息

方案 C: 搜索系统
- 搜索姓名/编制名/军功
- 快速定位特定单位
- 查看完整履历

方案 D: 时间线视图
- 按时间顺序显示重大事件
- 可筛选特定编制/战役
- 宏观视角看战争进程
```

**推荐方案**：A + B + C 组合
- 悬停预览：快速查看
- 编制面板：详细管理
- 搜索系统：精准定位

## 📊 更新后的核心机制

### 命令链（新增）

```
玩家 → 总部 → 传令兵 → 下级编制 → 执行

关键约束:
1. 玩家不能跳过编制层级直接指挥个体
2. 传令需要时间（距离/通讯手段）
3. 传令兵可能被拦截
4. 命令到达后才生效
```

### 恐慌与崩溃（更新）

```
个体恐慌 → 群体恐慌 → 凝聚力下降 → 群体崩溃 → 传染扩散

关键机制:
1. 恐慌在编制内快速传播
2. 凝聚力决定崩溃阈值
3. 崩溃如多米诺骨牌传染
4. 军官/老兵可稳定军心
```

## 🎮 游戏流程示例

### 场景：古代战争 - 指挥一个师

```
1. 玩家决策：命令第 1 团进攻敌方左翼

2. 命令下达:
   - 玩家点击第 1 团图标
   - 选择"进攻"命令
   - 指定目标（敌方左翼）

3. 传令过程:
   - 传令兵从总部出发（骑马）
   - 距离 500 米，速度 300 米/分钟
   - 预计 1 分 40 秒到达

4. 意外情况:
   - 传令兵途中遭遇敌方斥候
   - 被拦截，命令未能送达
   - 第 1 团未收到命令，原地待命

5. 玩家应对:
   - 发现传令兵失联
   - 派出第 2 个传令兵（绕路）
   - 或改用旗鼓号（但信息简单）

6. 命令生效:
   - 传令兵到达第 1 团总部
   - 团长接收命令
   - 第 1 团开始进攻

7. 战斗过程:
   - 第 1 团遭遇顽强抵抗
   - 伤亡增加，恐慌度上升
   - 凝聚力下降

8. 群体崩溃:
   - 某个连队率先溃散
   - 恐慌传播到邻近连队
   - 整个团开始崩溃

9. 玩家应对:
   - 投入预备队稳定战线
   - 或下达撤退命令（需要再次传令）
```
