## 🕵️ 斥候系统（战争迷雾精简版）

### 核心设计

**战争迷雾存在，但只对"上层建筑"生效**：

| 地图层 | 是否迷雾 | 说明 |
|--------|---------|------|
| **地形层** | ❌ 始终可见 | 山川/河流/湖泊（卫星数据） |
| **建筑层** | ✅ 需要侦察 | 建筑/道路/城市 |
| **敌军层** | ✅ 需要侦察 | 敌方部队位置 |

---

### 侦察机制

**情报获取流程**：
```
斥候探马出发
   ↓
发现目标（建筑/道路/敌军）
   ↓
安全返回总部（或无线电回报）
   ↓
情报传递成功
   ↓
地图上标注目标
```

**情报特点**：
- ✅ **准确性**：100% 准确（无假消息/间谍，简化复杂度）
- ⚠️ **时效性**：情报可能已过时
- ⚠️ **动态性**：敌军移动后需要重新标注

---

### 情报更新机制

**动态标注系统**：
```
时间 T1: 斥候 A 回报"敌军在位置 X"
        → 地图标注：位置 X 有敌军

时间 T2: 敌军实际已移动到位置 Y
        → 地图仍显示位置 X（情报过时）

时间 T3: 斥候 B 发现"敌军在位置 Y"
        → 地图更新：位置 Y 有敌军，清除位置 X 标注
```

**情报时效显示**：
```
地图标注格式:
  [敌军图标] 位置 Y
  最后更新：10 分钟前
  情报来源：侦察连第 2 排
  可信度：高（多斥候确认）
```

---

### 侦察部队编制

**玩家必须考虑配备侦察单位**：

#### 标准编制（推荐）
```
师级编制:
  ├─ 师直属侦察连 ✅
  ├─ 师直属通讯连 ✅
  ├─ 第 1 团
  ├─ 第 2 团
  └─ 第 3 团
```

#### 无侦察编制（不推荐）
```
师级编制:
  ├─ 第 1 团
  ├─ 第 2 团
  └─ 第 3 团
  
效果:
  ❌ 只能被动发现敌人
  ❌ 战场态势感知差
  ❌ 容易被伏击
```

### 视野系统

#### 双层视野机制

**战略层（迷雾层）**：
| 内容 | 可见性 | 说明 |
|------|--------|------|
| 地形 | ✅ 始终可见 | 山川/河流/湖泊 |
| 建筑 | ⚠️ 需侦察 | 城市/村庄/工厂 |
| 道路 | ⚠️ 需侦察 | 公路/铁路/桥梁 |
| 敌军 | ⚠️ 需侦察 | 敌方部队位置 |
| 友军 | ✅ 始终可见 | 己方部队位置 |

**战斗层（上帝视角）**：
- ✅ 交战时自动开启局部上帝视角
- ✅ 不影响玩家观赏战斗画面
- ✅ 仅用于战斗指挥

---

### 视野范围

| 单位类型 | 视野范围 | 时代限制 |
|---------|---------|---------|
| **步兵** | 1-2km | 所有时代 |
| **骑兵** | 3-5km | 古代/近代 |
| **侦察车** | 5-10km | 近代/现代 |
| **侦察机** | 50-100km | 现代 |
| **雷达** | 200-500km | 现代 |
| **卫星** | 全图 | 现代 |

---

### 通讯系统

**情报传递速度**：

| 通讯方式 | 传递速度 | 时代 |
|---------|---------|------|
| **传令兵（骑马）** | 距离÷速度 | 古代 |
| **旗鼓号** | 即时（视距内） | 古代 |
| **电话线** | 即时（需铺设） | 近代 |
| **电台** | 即时（无线） | 近代/现代 |
| **数据链** | 即时 + 可视化 | 现代 |

**通讯兵编制**：
```
通讯连职责:
  - 架设电话线
  - 操作电台
  - 维护通讯设备
  - 传递情报
  
无通讯连后果:
  - 情报传递慢
  - 指挥延迟高
  - 协同作战困难
```

## 🎮 玩家体验流程

### 典型侦察作战

```
1. 战前准备
   玩家决策:
   - 给第 X 师配备侦察连
   - 给第 X 师配备通讯连
   - 下达侦察命令

2. 侦察行动
   侦察连执行:
   - 分散搜索目标区域
   - 发现敌军第 Y 团
   - 记录位置/兵力/装备
   - 无线电回报总部

3. 情报标注
   玩家看到:
   - 地图上出现"敌军"标注
   - 位置：A 地区
   - 兵力：约 1 个团
   - 时间戳：5 分钟前
   - 可信度：高

4. 战略决策
   玩家根据情报:
   - 方案 A: 调动第 X 师前往 A 地区
   - 方案 B: 设伏等待敌军
   - 方案 C: 绕过 A 地区
   
   玩家选择方案 A

5. 部队移动
   第 X 师前往 A 地区:
   - 运输连自动安排车辆
   - 预计 2 小时到达

6. 情报更新/验证
   到达 A 地区后:
   - 情况 A: 敌军已离开（情报过时）
   - 情况 B: 发现敌军（情报准确）
   - 情况 C: 发现更多敌军（情报不完整）
   
   无论哪种情况，玩家都需要重新决策
```

## 📊 设计意义

### 战略深度

**侦察系统增加的决策点**：
- ✅ 是否配备侦察部队？（编制决策）
- ✅ 侦察部队派往哪里？（资源分配）
- ✅ 情报是否可信？（判断能力）
- ✅ 是否验证情报？（多斥候确认）
- ✅ 何时行动？（时机把握）

### 后勤重要性

**通讯兵的价值**：
- ✅ 情报传递速度 = 通讯兵质量
- ✅ 现代电台 vs 古代传令兵
- ✅ 通讯中断 = 情报无法送达 = 战场被动

### 战争迷雾精简

**只迷雾"上层建筑"**：
- ✅ 地形始终可见（不影响战略部署）
- ✅ 建筑/敌军需要侦察（增加探索乐趣）
- ✅ 情报有时效性（动态战场）
- ✅ 无假消息（简化复杂度）
