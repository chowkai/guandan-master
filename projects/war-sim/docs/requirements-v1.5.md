## 🎯 视角与精度系统

### 核心设计

**战斗精度 = 视角分辨率**：
- ✅ 根据视角缩放自动切换显示精度
- ✅ 根据编制密集度自动聚合/分散
- ✅ UI 支持树状列表快速选择定位

### 视角层级

| 层级 | 缩放比例 | 显示内容 | 性能 |
|------|---------|---------|------|
| **战略视角** | 1km+/像素 | 编制符号（团/营/连） | 极高 |
| **战役视角** | 100m/像素 | 营/连/排符号 + 密集度 | 高 |
| **战术视角** | 10m/像素 | 排/班 + 简单个体 | 中 |
| **战斗视角** | 1m/像素 | 个体士兵 + 动画 | 低 |

### 自动显隐规则

```
显示逻辑:
  IF 视角距离 > 1km:
    显示编制符号（不显示个体）
  ELSE IF 视角距离 > 100m:
    显示密集度色块
  ELSE IF 视角距离 > 10m:
    显示简化个体
  ELSE:
    显示完整个体 + 动画

编制密集度:
  IF 单位密度 > 100 个/屏:
    自动聚合为密集度色块
  ELSE:
    显示个体
```

### UI 树状列表

```
编制树状列表:
  ├─ 第 1 师
  │   ├─ 第 1 团
  │   │   ├─ 第 1 营
  │   │   │   ├─ 第 1 连
  │   │   │   └─ 第 2 连
  │   │   └─ 第 2 营
  │   └─ 第 2 团
  └─ 第 2 师

功能:
  - 快速搜索编制名
  - 点击快速定位（镜头跳转）
  - 显示编制状态（兵力/士气/补给）
  - 批量选择（Ctrl+ 点击）
```

## 🤖 AI 自主系统

### 核心设计理念

**玩家只关注战略，AI 自动执行细节**：

> "编制下的每个个体都要根据自身的职责去自发有条不紊地去工作。"

### AI 层级

| AI 类型 | 自主程度 | 玩家干预 | 示例 |
|--------|---------|---------|------|
| **后勤 AI** | 完全自主 | 无需干预 | 运输/补给/维修 |
| **战术 AI** | 半自主 | 可干预 | 战斗/移动/防御 |
| **个体 AI** | 半自主 | 不可直接干预 | 生存/执行职责 |

---

### 后勤 AI（完全自主）

**职责**：
- ✅ 自动安排运输
- ✅ 自动分配补给
- ✅ 自动维修装备
- ✅ 自动医疗救治

**运输连自动调度案例**：

```
玩家指令:
  "第 X 团移动 150 公里去驻防"

运输连 AI 自动执行:
  1. 评估需求
     - 士兵：2000 人
     - 辎重：50 吨
     - 距离：150km
  
  2. 安排运力
     - 调用卡车×30 辆
     - 安排司机×60 人（轮班）
     - 计算燃油：30 辆×150km×0.3L/km = 1350L
  
  3. 规划路线
     - 选择安全路线（避开敌方）
     - 计算时间：150km ÷ 40km/h = 3.75 小时
     - 安排中途休息点
  
  4. 执行运输
     - 士兵集结上车
     - 辎重装载
     - 车队出发
  
  5. 到达后
     - 士兵下车
     - 辎重卸载
     - 车辆返回或待命

玩家看到的结果:
  "第 X 团已到达目的地，耗时 4 小时"
```

**玩家不需要操心**：
- ❌ 具体派哪辆车
- ❌ 司机怎么排班
- ❌ 路线怎么走
- ❌ 燃油够不够
- ❌ 中途怎么休息

**玩家只需要决策**：
- ✅ 是否移动该团
- ✅ 移动到哪里
- ✅ 何时出发
- ✅ 是否中途停止

---

### 战术 AI（半自主）

**职责**：
- ✅ 自动执行战斗命令
- ✅ 自动选择掩体
- ✅ 自动火力覆盖
- ✅ 自动撤退（如果崩溃）

**玩家可干预**：
- ✅ 改变攻击目标
- ✅ 调整进攻路线
- ✅ 下达撤退命令
- ✅ 请求火力支援

---

### 个体 AI（半自主）

**职责**：
- ✅ 自动执行上级命令
- ✅ 自动寻找掩体
- ✅ 自动补充弹药
- ✅ 自动救治伤员

**生存优先级**：
```
第一原则：确保生存
  - 躲避炮火
  - 寻找掩护
  - 逃跑/投降（如果绝望）

第二原则：执行职责
  - 士兵：射击/移动
  - 医护兵：救治伤员
  - 运输兵：运送物资
  - 炊事兵：准备食物

第三原则：协助战友
  - 火力掩护
  - 搬运伤员
  - 共享弹药
```

## 🎮 玩家体验设计

### 典型游戏流程

```
1. 战略部署（战略视角）
   玩家在地图上看到各编制符号
   UI 显示：第 X 团（2000 人，士气 80，补给充足）
   
   玩家操作:
   - 点击第 X 团
   - 选择"移动"命令
   - 在地图上点击目标位置
   - 确认命令

2. 传令兵出发（自动动画）
   玩家看到:
   - 传令兵从总部出发
   - 前往第 X 团指挥部
   - 预计 10 分钟到达

3. 命令生效（AI 自动执行）
   传令兵到达后:
   - 第 X 团收到命令
   - 运输连自动安排车辆（无需玩家操心）
   - 士兵自动集结
   - 辎重自动装载
   
   玩家看到:
   - 第 X 团状态变为"移动中"
   - 地图上显示移动箭头
   - 预计 4 小时到达

4. 途中观察（可切换视角）
   玩家可随时切换:
   
   战略视角:
   - 看到移动箭头
   - 看到其他部队动态
   
   战术视角:
   - 看到车队行进
   - 看到行军序列
   
   战斗视角:
   - 看到士兵在车上
   - 看到司机驾驶
   - 听到士兵对话

5. 到达驻防（AI 自动完成）
   到达后:
   - 士兵自动下车
   - 辎重自动卸载
   - 自动建立防御工事
   - 自动派出哨兵
   
   玩家收到报告:
   "第 X 团已到达目的地
    耗时：4 小时
    状态：良好
    防御工事：已建立"

6. 继续战略决策
   玩家下达下一个命令
   循环继续
```

## 📊 设计意义

### 玩家价值

**玩家专注于**：
- ✅ 战略规划（打哪里）
- ✅ 编制部署（怎么排兵布阵）
- ✅ 资源分配（优先补给谁）
- ✅ 时机把握（何时进攻/撤退）

**AI 负责**：
- ✅ 后勤执行（运输/补给）
- ✅ 战术执行（战斗/移动）
- ✅ 个体行为（射击/躲避）

### 游戏特色

**差异化优势**：
- ✅ 真实后勤（自动但存在）
- ✅ 真实指挥链（传令兵延迟）
- ✅ 真实个体（有自主性）
- ✅ 无缝视角（战略到战术）

**不是**：
- ❌ 微操 RTS（每个兵都要管）
- ❌ 抽象数字（看不到个体）
- ❌ 瞬间移动（需要运输时间）
