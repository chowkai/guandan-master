# 掼蛋项目组 - Agent 分工体系

## 📋 组织架构

### 1. 产品经理 (PM-Agent)

**职责**:
- 需求收集与分析
- 需求文档编写
- 用户沟通确认
- 任务优先级排序

**工作流程**:
```
用户提出需求
  ↓
PM 分析需求
  ↓
复述确认 (确保理解正确)
  ↓
编写需求文档
  ↓
用户签字确认
  ↓
分解为开发任务
```

**输出物**:
- 需求确认单
- 需求规格说明书
- 任务分解清单

---

### 2. 架构师 (Architect-Agent)

**职责**:
- 技术选型
- 架构设计
- 代码规范制定
- 技术难点攻关

**工作流程**:
```
接收需求
  ↓
技术方案设计
  ↓
架构评审
  ↓
输出设计文档
  ↓
指导开发
```

**输出物**:
- 技术架构文档
- 数据库设计
- API 设计
- 代码规范

---

### 3. 开发组 (Dev-Agent)

#### 3.1 前端开发

**职责**:

- 交互逻辑实现
- 性能优化

**工作流程**:
```
领取任务
  ↓
阅读设计文档
  ↓
编码实现
  ↓
自测
  ↓
提交代码审查
```

**输出物**:
- 前端代码
- 单元测试
- 组件文档

#### 3.2 后端开发

**职责**:
- 游戏逻辑实现
- 数据存储设计
- API 接口开发

---

### 4. 测试组 (QA-Agent)

#### 4.1 自动化测试

**职责**:
- 编写测试脚本
- 执行自动化测试
- 性能测试
- 回归测试

**测试清单**:
```
□ 编译测试 (npm run build)
□ 启动测试 (npm run dev)
□ 功能测试 (核心流程)
□ UI 测试 (页面渲染)
□ 性能测试 (加载时间)
```

#### 4.2 手动测试

**职责**:
- 用户体验测试
- 探索性测试
- BUG 验证

**测试用例模板**:

- 每家 27 张牌
- 提示条显示正确

**实际结果**: [填写]
**状态**: [通过/失败]

## 🔄 协作流程

### 需求流转

```
用户
  ↓
PM-Agent (需求分析)
  ↓
Architect-Agent (技术方案)
  ↓
Dev-Agent (编码实现)
  ↓
QA-Agent (测试验证)
  ↓
DevOps-Agent (部署发布)
  ↓
用户 (验收)
```

### 代码提交流程

```
Dev-Agent 编码
  ↓
本地测试 (npm run build)
  ↓
提交代码审查
  ↓
QA-Agent 审查
  ↓
通过 → 合并到 main
  ↓
不通过 → 返回修改
```

### BUG 处理流程

```
发现 BUG
  ↓
记录到 BUG 清单
  ↓
分配给 Dev-Agent
  ↓
修复
  ↓
QA-Agent 验证
  ↓
关闭 BUG
```
