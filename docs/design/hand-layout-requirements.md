## 🎯 Phase 3 实施计划

### 3.5 UI 实现 - 手牌区布局

**任务**: 手牌区网格布局实现  
**负责人**: frontend-agent  
**优先级**: P0

#### 技术要求

**布局规格**:
- 列数：5 列（固定）
- 行数：5 行（默认，未来自选）
- 卡牌尺寸：80×112px（1:1.4 比例）
- 牌间距：待测算（建议 8-12px）

**牌间距测算公式**:
```
可用宽度 = 屏幕宽度 - 左右边距（750px - 40px = 710px）
列宽 = 卡牌宽度 + 牌间距（80px + gap）
5 列总宽 = 5 × (80 + gap) ≤ 710px
gap ≤ 62px

可用高度 = 手牌区高度 - 上下边距（572px - 32px = 540px）
行高 = 卡牌高度 + 行间距（112px + rowGap）
5 行总高 = 5 × (112 + rowGap) ≤ 540px
rowGap ≤ -4px（需要调整）

优化方案:
- 方案 1: 缩小卡牌尺寸（70×98px，保持 1:1.4）
- 方案 2: 增加手牌区高度（600px）
- 方案 3: 减少列数（4 列×7 行）

推荐：方案 1（缩小卡牌至 70×98px）
```

**推荐规格**:
- 卡牌尺寸：70×98px（1:1.4 比例）
- 牌间距：10px
- 5 列总宽：5 × (70 + 10) = 400px（居中显示，左右留白）
- 5 行总高：5 × (98 + 8) = 530px（符合手牌区高度）

#### 未来自选行数功能

**需求**: 支持用户自定义行数（满足不同理牌习惯）

**选项**:
| 行数 | 列数 | 总牌数 | 适用场景 |
|------|------|--------|----------|
| 3 行 | 9 列 | 27 张 | 喜欢紧凑布局 |
| 4 行 | 7 列 | 28 张 | 平衡型 |
| 5 行 | 6 列 | 30 张 | **默认推荐** |
| 6 行 | 5 列 | 30 张 | 喜欢宽松布局 |

**实现方式**:
- 设置页面增加"手牌布局"选项
- 支持 3-6 行切换
- 自动计算最佳列数和牌间距
- 保存用户偏好（本地存储）
