## 1. 竖屏空间分析

- **设备**: 微信小程序竖屏模式

## 2. 手牌区 27 张牌排列方案对比

### 2.1 设计约束
- **最大手牌数**: 27 张（掼蛋规则上限）
- **单张牌尺寸**: 80px 宽 × 120px 高
- **牌间距**: 8px
- **操作要求**: 竖屏单手友好，轻松选牌、出牌

### 2.2 方案 A: 单列排列

```
计算:
  宽度需求 = 27 张 × (80px + 8px) = 2376px
  屏幕宽度 = 750px
  结果 = 2376px > 750px ❌ 超出屏幕 3.17 倍
```

**结论**: ❌ **不可行** - 严重超出屏幕宽度

### 2.3 方案 B: 两列排列（推荐）

```
计算:
  每列牌数 = ceil(27/2) = 14 张（左列）+ 13 张（右列）
  单列宽度 = 14 张 × (80px + 8px) = 1232px（横向滚动）
  单列高度 = 14 张 × 180px = 2520px（纵向滚动）
  
  优化后（横向并排两列）:
  总宽度 = 2 × 80px + 8px 间距 = 168px
  总高度 = 14 × (120px + 8px) = 1792px
```

**布局示意**:
```
┌─────────────────────────────────┐
│  左列 (14 张)    │    右列 (13 张)   │
│  ┌────┐        │        ┌────┐  │
│  │ ♠A │        │        │ ♥K │  │
│  ├────┤        │        ├────┤  │
│  │ ♠K │        │        │ ♥Q │  │
│  ├────┤        │        ├────┤  │
│  │ ...│        │        │ ...│  │
│  ├────┤        │        ├────┤  │
│  │ ♠2 │        │        │ ♥2 │  │
│  └────┘        │        └────┘  │
└─────────────────────────────────┘
     168px 宽           168px 宽
       ↓                  ↓
    1792px 高          1792px 高
```

**优点**:
- ✅ 单手友好（左手模式/右手模式可切换）
- ✅ 牌面大，易于识别
- ✅ 支持滑动选牌
- ✅ 符合用户建议

**缺点**:
- ⚠️ 需要纵向滑动（27 张牌需滚动）
- ⚠️ 占用高度较大（1792px > 可用 1246px）

### 2.4 方案 C: 三列排列

```
计算:
  每列牌数 = ceil(27/3) = 9 张
  总宽度 = 3 × 80px + 2 × 8px 间距 = 256px
  总高度 = 9 × (120px + 8px) = 1152px
```

**优点**:
- ✅ 占用空间小（256px 宽 × 1152px 高）
- ✅ 可容纳在屏幕内

**缺点**:
- ❌ 单手操作困难（三列跨越屏幕，需双手）
- ❌ 牌面密集，易误触
- ❌ 不符合竖屏单手操作原则

### 2.5 手牌区方案对比总结

| 方案 | 宽度 | 高度 | 单手友好 | 推荐度 |
|------|------|------|----------|--------|
| A: 单列 | 2376px ❌ | 180px | ❌ | ⭐ |
| B: 两列 | 168px ✅ | 1792px | ✅ | ⭐⭐⭐⭐⭐ |
| C: 三列 | 256px ✅ | 1152px | ❌ | ⭐⭐ |

**🏆 推荐方案**: **方案 B（两列排列）**

**理由**:
1. 符合用户明确建议（"排两列或三列"）
2. 单手操作友好，支持左右手模式
3. 牌面大，易于识别和选择
4. 虽然需要滚动，但掼蛋出牌频率低，可接受

## 3. 出牌区 4 组牌布局方案对比

### 3.1 设计约束
- **玩家数量**: 4 家（对家、上家、下家、自己）
- **每组最大牌数**: 10 张（8 张核弹 + 2 张红心）
- **常见牌数**: 6 张一组
- **单张牌尺寸**: 60px 宽 × 90px 高（出牌区缩小）
- **牌间距**: 4px

### 3.2 方案 A: 2×2 网格（推荐）

```
布局:
┌─────────────────────────────────┐
│     对家 (北)                    │
│     [牌组 1]                     │
├──────────────┬──────────────────┤
│   上家 (西)   │    下家 (东)      │
│   [牌组 2]   │    [牌组 3]      │
├──────────────┴──────────────────┤
│     自己 (南)                    │
│     [牌组 4]                     │
└─────────────────────────────────┘

尺寸计算:
  单组宽度 = 6 张 × (60px + 4px) = 384px
  单组高度 = 90px + 20px 边距 = 110px
  2×2 网格总尺寸 = 384px × 2 + 间距 = 800px 宽
                   110px × 2 + 间距 = 260px 高
```

**优点**:
- ✅ 对称美观，符合传统牌桌布局
- ✅ 四家清晰区分
- ✅ 中央区域可展示当前出牌
- ✅ 符合用户视觉习惯

**缺点**:
- ⚠️ 占用中央空间（可能与手牌区冲突）
- ⚠️ 需要优化布局避免遮挡

### 3.3 方案 B: 横向 4 栏

```
布局:
┌─────────────────────────────────┐
│ [对家] [上家] [下家] [自己]      │
│  牌组   牌组   牌组   牌组       │
└─────────────────────────────────┘

尺寸计算:
  单栏宽度 = 750px / 4 = 187.5px
  单栏可容纳牌数 = 187.5 / 64px ≈ 2-3 张 ❌
```

**优点**:
- ✅ 清晰区分四家

**缺点**:
- ❌ 严重压缩手牌空间
- ❌ 每组牌显示不全（最多 2-3 张）
- ❌ 不符合实际出牌展示需求

### 3.4 方案 C: 折叠式

```
布局:
默认状态:
┌─────────────────────────────────┐
│  当前出牌玩家：对家              │
│  [牌组展示]                     │
│  [展开查看全部 ▼]               │
└─────────────────────────────────┘

展开状态:
┌─────────────────────────────────┐
│  对家 [牌组]                    │
│  上家 [牌组]                    │
│  下家 [牌组]                    │
│  自己 [牌组]                    │
└─────────────────────────────────┘
```

**优点**:
- ✅ 节省空间
- ✅ 默认状态简洁

**缺点**:
- ❌ 交互复杂（需要点击展开）
- ❌ 无法同时查看四家出牌
- ❌ 掼蛋需要记忆各家出牌，折叠不便

### 3.5 出牌区方案对比总结

| 方案 | 空间占用 | 信息完整 | 交互复杂度 | 推荐度 |
|------|----------|----------|------------|--------|
| A: 2×2 网格 | 中等 | ✅ 完整 | 低 | ⭐⭐⭐⭐⭐ |
| B: 横向 4 栏 | 大 | ❌ 不完整 | 低 | ⭐ |
| C: 折叠式 | 小 | ⚠️ 需展开 | 高 | ⭐⭐ |

**🏆 推荐方案**: **方案 A（2×2 网格，优化布局）**

**优化建议**:
1. 将 2×2 网格放置在屏幕中上部
2. 缩小出牌区牌面尺寸（60px vs 手牌区 80px）
3. 当前出牌玩家牌组高亮显示
4. 已出牌组半透明处理，降低视觉干扰

## 4. 玩家区域简化方案

### 4.1 设计原则
- **极简主义**: 只保留必要信息
- **动态提示**: 关键状态才显示
- **视觉清晰**: 头像 + 名称即可识别

### 4.2 布局方案

```
┌─────────────────────────────────┐
│  ⏸️ 暂停    第 3 局              │  ← 顶部状态栏
├─────────────────────────────────┤
│                                 │
│        ┌─────────┐              │
│        │  对家   │ 剩 6 张 🔔    │  ← 剩 6 张及以下显示角标
│        │  头像   │              │
│        │  张三   │              │
│        └─────────┘              │
│                                 │
├───────────────┬─────────────────┤
│   ┌─────┐     │     ┌─────┐    │
│   │ 上家 │     │     │ 下家 │    │
│   │ 头像 │     │     │ 头像 │    │
│   │ 李四 │     │     │ 王五 │    │
│   └─────┘     │     └─────┘    │
├───────────────┴─────────────────┤
│        ┌─────────┐              │
│        │  自己   │              │
│        │  头像   │              │
│        │  我     │              │
│        └─────────┘              │
└─────────────────────────────────┘
```

### 4.3 元素规格

| 元素 | 尺寸 | 位置 | 说明 |
|------|------|------|------|
| 暂停按钮 | 44px × 44px | 左上角 | 点击暂停游戏 |
| 局数显示 | 自适应 | 顶部中央 | "第 X 局" |
| 玩家头像 | 60px × 60px | 四角 | 圆形头像 |
| 玩家名称 | 自适应 | 头像下方 | 最多 4 字 |
| 剩牌角标 | 24px × 24px | 头像右上角 | 仅≤6 张显示 🔔 |

### 4.4 AI 头像规范
- **默认头像**: 卡通风格 AI 形象
- **颜色区分**: 四家使用不同底色（蓝/绿/橙/紫）
- **表情状态**: 可选（思考中/已出牌/胜利）

## 5. 推荐方案汇总

### 5.1 最终推荐

| 区域 | 推荐方案 | 关键参数 |
|------|----------|----------|
| **手牌区** | 两列排列 | 14 张/列，168px 宽，支持滚动 |
| **出牌区** | 2×2 网格 | 60px 牌面，对称布局 |
| **玩家区** | 简化头像 + 名称 | 60px 头像，≤6 张角标提醒 |

### 5.2 推荐理由

1. **符合用户反馈**: 直接响应用户提出的"两列或三列"建议
2. **单手友好**: 竖屏操作符合人体工学
3. **信息完整**: 四家出牌同时可见，便于记忆和策略
4. **视觉简洁**: 去除冗余信息，聚焦核心玩法
5. **可扩展性**: 方案支持后续功能迭代（如记牌器、提示等）
