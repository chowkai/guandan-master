## 🔄 固化工作流程 (跨项目通用)

### 流程 1: 需求管理 (所有项目必须执行)

```
┌─────────────────────────────────────────────────────────┐
│ 需求管理流程 (固化)                                      │
├─────────────────────────────────────────────────────────┤
│                                                          │
│  用户提出需求                                            │
│       ↓                                                  │
│  PM-Agent 分析并复述                                      │
│       ↓                                                  │
│  用户确认 (签字)                                          │
│       ↓                                                  │
│  写入需求文档 (docs/requirements/REQ-YYYY-XXX.md)        │
│       ↓                                                  │
│  分解任务 (docs/tasks/TASK-YYYY-XXX.md)                  │
│       ↓                                                  │
│  分配给对应 Agent                                         │
│                                                          │
└─────────────────────────────────────────────────────────┘
```

### 流程 2: 开发流程 (所有项目必须执行)

```
┌─────────────────────────────────────────────────────────┐
│ 开发流程 (固化)                                          │
├─────────────────────────────────────────────────────────┤
│                                                          │
│  领取任务                                                │
│       ↓                                                  │
│  阅读需求文档 + 设计文档                                  │
│       ↓                                                  │
│  本地编码                                                │
│       ↓                                                  │
│  自检清单 (见 CODE-CHECKLIST.md)                         │
│       ↓                                                  │
│  提交审查 (git commit)                                   │
│       ↓                                                  │
│  QA-Agent 审查                                           │
│       ↓                                                  │
│  通过 → 合并到 main                                      │
│  不通过 → 返回修改                                        │
│                                                          │
└─────────────────────────────────────────────────────────┘
```

### 流程 3: 测试流程 (所有项目必须执行)

```
┌─────────────────────────────────────────────────────────┐
│ 测试流程 (固化)                                          │
├─────────────────────────────────────────────────────────┤
│                                                          │
│  编译测试 (npm run build)                                │
│       ↓                                                  │
│  启动测试 (npm run dev)                                  │
│       ↓                                                  │
│  功能测试 (核心流程)                                      │
│       ↓                                                  │
│  UI 测试 (页面渲染)                                       │
│       ↓                                                  │
│  性能测试 (加载时间/帧率)                                 │
│       ↓                                                  │
│  生成测试报告 (docs/test/TEST-REPORT-XXX.md)            │
│       ↓                                                  │
│  通过 → 发布                                             │
│  失败 → 提 BUG → 修复 → 回归测试                          │
│                                                          │
└─────────────────────────────────────────────────────────┘
```

### 流程 4: 发布流程 (所有项目必须执行)

```
┌─────────────────────────────────────────────────────────┐
│ 发布流程 (固化)                                          │
├─────────────────────────────────────────────────────────┤
│                                                          │
│  版本打包 (npm run build)                                │
│       ↓                                                  │
│  更新版本号 (package.json)                               │
│       ↓                                                  │
│  编写发布说明 (docs/release/RELEASE-vX.X.X.md)          │
│       ↓                                                  │
│  git commit + git push                                   │
│       ↓                                                  │
│  部署到服务器                                            │
│       ↓                                                  │
│  验证运行                                                │
│       ↓                                                  │
│  用户验收                                                │
│                                                          │
└─────────────────────────────────────────────────────────┘
```

```
代码提交前必须通过:

任何一项失败 → 拒绝合并
```

## 🎯 行为准则 (固化到每次交互)

### 每次收到需求的处理流程

```
1. PM-Agent 复述需求
   "理解您的需求是：[复述内容]，请问是否正确？"

2. 用户确认后
   "已记录到需求文档：docs/requirements/REQ-2026-XXX.md"

3. 分解任务
   "已分解为以下任务：[任务列表]"

4. 分配执行
   "已分配给 [Agent 名称]，预计 [时间] 完成"

5. 完成后
   "任务已完成，测试报告：docs/test/TEST-XXX.md"

6. 用户验收
   "请验收，如有问题请指出"
```

### 每次提交的检查清单

```
提交前必须执行:
1. npm run build (编译检查)
2. 检查 Console 错误
3. 核心功能测试
4. 填写提交检查单
5. QA-Agent 审查

任何一项未完成 → 禁止提交
```

### 每日工作结束前

```
1. 更新任务追踪表
2. 编写工作日志
3. 更新项目记忆库
4. 列出明日计划
```
