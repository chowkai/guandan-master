## 1. 架构总览

### 1.1 架构原则

1. **本地优先** - 单机模式完全本地运行，无需网络
2. **双模式支持** - 单机 / P2P 可切换，代码复用
3. **轻量通信** - P2P 模式仅信令经过服务器，数据直连
4. **无状态设计** - 信令服务器不存储游戏状态
5. **断线友好** - 支持断线重连，房主托管状态

## 4. 通信协议设计

### 4.1 信令协议（WebSocket）

#### 4.1.1 消息格式

```typescript
interface SignalingMessage {
  type: MessageType;
  roomId: string;
  from: string;      // 发送者 ID
  to?: string;       // 接收者 ID（可选，广播时为空）
  payload: any;      // 消息内容
  timestamp: number; // 时间戳
}

type MessageType =
  | 'create-room'       // 创建房间
  | 'join-room'         // 加入房间
  | 'room-created'      // 房间已创建（服务器响应）
  | 'room-joined'       // 房间已加入（服务器响应）
  | 'player-joined'     // 有玩家加入（广播）
  | 'offer'             // WebRTC Offer
  | 'answer'            // WebRTC Answer
  | 'ice-candidate'     // ICE Candidate
  | 'heartbeat'         // 心跳
  | 'error';            // 错误
```

#### 4.1.2 通信流程

```
房主创建房间:
  Client ──create-room──> Server
  Client <──room-created── Server

玩家加入房间:
  Client ──join-room(inviteCode)──> Server
  Client <──room-joined── Server
  Server ──player-joined──> 房主

交换 WebRTC 信令:
  房主 ──offer──> Server ──offer──> 玩家
  玩家 ──answer──> Server ──answer──> 房主
  双方 ──ice-candidate──> Server ──ice-candidate──> 对方

P2P 连接建立后:
  房主 <══════════WebRTC DataChannel══════════> 玩家
  （信令服务器不再参与）
```

### 4.2 游戏数据协议（WebRTC DataChannel）

```typescript

type GameDataType =
```

## 8. 安全设计

### 8.1 通信安全

- ✅ WebRTC 使用 DTLS/SRTP 加密
- ✅ WebSocket 使用 WSS（HTTPS）
- ✅ 信令服务器不存储敏感数据
- ✅ 房间邀请码随机生成（防猜测）

### 8.2 数据安全

- ✅ 本地存储加密（可选）
- ✅ 不收集用户隐私数据
- ✅ 游戏记录本地保存
- ✅ 无账号系统（无需密码）

### 8.3 反作弊

- ⚠️ P2P 模式房主可作弊（技术限制）
- ✅ 单机模式无作弊动机
- ✅ 无排行榜/无奖励（降低作弊动机）
- ✅ 产品定位为休闲，非竞技
