## 🎯 使用原则

### 1. 辅助不替代

```
✅ AI 辅助编程
❌ AI 替代思考

AI 是助手，不是决策者
```

### 2. 审查必执行

```
AI 生成代码
  ↓
人工审查 (必须)
  ↓
测试验证 (必须)
  ↓
代码审查 (必须)
  ↓
合并
```

### 3. 安全第一条

```
❌ 禁止上传核心代码
❌ 禁止上传敏感配置
❌ 禁止上传用户数据
✅ 可以上传通用代码
```

## 🔧 使用流程

### 流程 1: 代码审查流程

```
1. 完成编码
   ↓
2. 运行 claude-code review
   ↓
3. 修复 AI 提出的问题
   ↓
4. 再次审查
   ↓
5. 提交代码
```

**示例**:
```bash
# 审查当前文件
claude-code "审查 src/components/GameTable.svelte"

# 修复建议:
- 第 23 行：未使用的变量
- 第 45 行：潜在的空指针
- 第 67 行：可以优化性能

# 修复后再次审查
claude-code "再次审查，确认修复"
```

---

### 流程 2: Bug 修复流程

```
1. 发现 bug
   ↓
2. 描述 bug 给 AI
   ↓
3. AI 分析原因
   ↓
4. AI 提供修复方案
   ↓
5. 人工审查方案
   ↓
6. 应用修复
   ↓
7. 测试验证
```

**示例**:
```bash
# 描述 bug
claude-code "bug: AI 出牌后游戏卡住，不切换到下一回合"

# AI 分析:
# 原因：nextTurn() 中状态更新和 AI 触发顺序问题
# 修复：先更新状态，再触发 AI

# 应用修复
# 人工审查代码
# 测试验证
```

---

### 流程 3: 测试生成流程

```
1. 完成功能开发
   ↓
2. 请求 AI 生成测试
   ↓
3. 审查测试用例
   ↓
4. 运行测试
   ↓
5. 修复失败的测试
```

**示例**:
```bash
# 生成测试
claude-code "为 game.ts 的 playCards() 函数生成测试"

# AI 生成:
# - 正常出牌测试
# - 无效牌型测试
# - 管牌验证测试
# - 边界条件测试

# 审查并运行
npm test
```
