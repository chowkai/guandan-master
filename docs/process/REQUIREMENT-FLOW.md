# 需求管理流程 (固化版)

**适用范围**: 所有项目  
**版本**: v1.0  
**生效日期**: 2026-03-21

---

## 🎯 流程目标

确保需求：
1. 被正确理解
2. 被完整记录
3. 不被遗忘
4. 可追溯验收

---

## 📋 需求处理流程

### 步骤 1: 需求收集

**触发**: 用户提出需求

**执行**: PM-Agent

**动作**:
```
1. 记录用户原话
2. 记录时间
3. 记录上下文
```

**输出**: 需求原始记录

---

### 步骤 2: 需求分析

**执行**: PM-Agent

**动作**:
```
1. 分析需求内容
2. 识别影响范围
3. 评估优先级
4. 初步估算工时
```

**输出**: 需求分析草稿

---

### 步骤 3: 需求复述 (关键步骤)

**执行**: PM-Agent

**动作**:
```
向用户复述:
"理解您的需求是：
1. [需求点 1]
2. [需求点 2]
3. [需求点 3]

验收标准:
- [标准 1]
- [标准 2]

请问理解是否正确？"
```

**目的**: 确保理解正确

---

### 步骤 4: 用户确认

**执行**: 用户

**动作**:
```
用户确认:
[ ] 理解正确
[ ] 需要修改

修改意见:
_________________
```

**输出**: 用户签字确认

---

### 步骤 5: 需求文档化

**执行**: PM-Agent

**动作**:
```
创建需求文档:
docs/requirements/REQ-YYYY-NNN.md

内容:
- 需求编号
- 提出时间
- 提出人
- 需求描述
- 验收标准
- 影响范围
- 优先级
- 预计工时
```

**输出**: 正式需求文档

---

### 步骤 6: 任务分解

**执行**: PM-Agent + Architect-Agent

**动作**:
```
分解为开发任务:
docs/tasks/TASK-YYYY-NNN.md

每个任务包含:
- 任务描述
- 技术方案
- 预计工时
- 负责人
- 截止日期
```

**输出**: 任务清单

---

### 步骤 7: 任务分配

**执行**: PM-Agent

**动作**:
```
分配给对应 Agent:
- UI 需求 → UI-Agent
- 前端需求 → Dev-Frontend-Agent
- 后端需求 → Dev-Backend-Agent
- 测试需求 → QA-Agent
```

**输出**: 任务分配表

---

### 步骤 8: 任务执行

**执行**: 对应 Agent

**动作**:
```
按开发流程执行:
1. 阅读需求文档
2. 技术方案设计
3. 编码实现
4. 自测
5. 提交审查
```

**输出**: 完成的任务

---

### 步骤 9: 测试验证

**执行**: QA-Agent

**动作**:
```
按测试流程执行:
1. 编译测试
2. 功能测试
3. 性能测试
4. 生成测试报告
```

**输出**: 测试报告

---

### 步骤 10: 用户验收

**执行**: 用户 + PM-Agent

**动作**:
```
用户验收:
[ ] 通过
[ ] 需修改

验收意见:
_________________
```

**输出**: 验收报告

---

### 步骤 11: 需求归档

**执行**: PM-Agent

**动作**:
```
更新需求状态:
- 标记为"已完成"
- 关联测试报告
- 关联发布版本
- 归档到记忆库
```

**输出**: 归档的需求

---

## 📊 需求追踪表

**位置**: `docs/requirements/TRACKER.md`

**格式**:
```markdown
| 编号 | 提出日期 | 需求描述 | 状态 | 负责人 | 完成日期 |
|------|---------|---------|------|--------|---------|
| REQ-001 | 2026-03-21 | 牌放大 | ✅ | Dev-Agent | 2026-03-21 |
| REQ-002 | 2026-03-21 | AI 快速 | ✅ | Dev-Agent | 2026-03-21 |
```

---

## 🧠 记忆机制

### 1. 需求记忆库

**位置**: `memory/project-memory.md`

**内容**:
```markdown
## 核心需求 (永久记忆)
1. [需求 1]
2. [需求 2]
3. [需求 3]

## 已完成需求
- [x] REQ-001: 牌放大 (2026-03-21)
- [x] REQ-002: AI 快速 (2026-03-21)

## 待办需求
- [ ] REQ-003: [描述]
```

### 2. 每日回顾

**时间**: 每天工作结束前

**内容**:
```
1. 回顾当天需求
2. 更新任务状态
3. 更新记忆库
4. 列出明日计划
```

### 3. 项目启动时

**动作**:
```
1. 阅读 memory/project-memory.md
2. 了解所有需求
3. 检查待办任务
4. 制定工作计划
```

---

## ⚠️ 违规处理

**不执行需求流程**:
- 第一次：警告 + 补全流程
- 第二次：严重警告 + 重新培训
- 第三次：禁止处理需求

**跳过用户确认**:
- 直接视为严重违规
- 重新执行完整流程
- 记录到质量报告

---

## 📈 质量指标

| 指标 | 目标值 | 测量方式 |
|------|--------|---------|
| 需求理解准确率 | 100% | 用户确认 |
| 需求文档完整率 | 100% | 文档检查 |
| 需求完成率 | >90% | 任务追踪 |
| 用户满意度 | >90% | 用户反馈 |

---

**此流程必须固化到每次需求处理，禁止跳过！**
