# 掼蛋大师 v3 - 需求调研计划

**阶段**: Phase 1: 需求调研  
**负责人**: 产品经理 (AI) + 周凯  
**预计时间**: 2-3 天  
**开始日期**: 2026-03-22

---

## 📋 调研目标

1. **理解用户真实需求** - 不是表面功能，而是深层体验
2. **竞品分析** - 学习优秀掼蛋游戏的优点
3. **明确产品定位** - 做什么，不做什么
4. **输出需求文档** - PRD v1.0

---

## 🎯 调研内容

### 1. 用户访谈 (周凯)

**时间**: 30-60 分钟  
**形式**: 对话访谈  
**目标**: 理解用户真实想法

**访谈提纲**:

#### A. 使用场景
- 什么时候玩掼蛋？（碎片时间/专门娱乐）
- 一个人玩还是多人？（单机/联机）
- 每局多长时间？（5 分钟/15 分钟/30 分钟+）
- 什么场景下会打开游戏？（等车/休息/睡前）

#### B. 痛点分析
- 之前玩的掼蛋游戏有什么不爽的？
- 哪些功能觉得多余？
- 哪些操作觉得麻烦？
- 哪些体验觉得差？

#### C. 期望体验
- 理想中的掼蛋游戏是什么样的？
- 最看重什么？（画面/速度/AI 智能/规则完整）
- 哪些功能是必须的？
- 哪些功能可以没有？

#### D. 竞品参考
- 玩过哪些掼蛋游戏？
- 最喜欢哪个？为什么？
- 最不喜欢哪个？为什么？
- 有哪些设计值得学习？

#### E. 视觉偏好
- 喜欢什么风格？（中国风/现代简约/卡通）
- 喜欢什么颜色？（深绿牌桌/其他）
- 牌面喜欢大还是小？
- 动画需要多还是少？

#### F. AI 期望
- AI 应该像真人还是像机器？
- 需要提示功能吗？
- 需要悔棋功能吗？
- 需要教学功能吗？

---

### 2. 竞品分析 (产品经理)

**目标**: 分析 3-5 款主流掼蛋游戏

**分析维度**:

| 维度 | 分析内容 |
|------|----------|
| **核心玩法** | 规则完整性、特殊牌型 |
| **用户体验** | 操作流畅度、提示清晰度 |
| **界面设计** | 视觉风格、色彩搭配、布局 |
| **AI 表现** | 出牌速度、智能程度 |
| **功能特性** | 特色功能、创新点 |
| **性能表现** | 加载速度、流畅度 |

**竞品候选**:
- 腾讯掼蛋
- JJ 掼蛋
- 掼蛋之家
- 其他热门掼蛋 APP

---

### 3. 需求整理 (产品经理)

**输出物**:

1. **用户画像** - 目标用户特征
2. **用户故事** - As a... I want... So that...
3. **功能清单** - 必须有/应该有/可以有
4. **优先级排序** - P0/P1/P2
5. **验收标准** - 每个功能的完成定义

---

## 📝 需求文档结构 (PRD v1.0)

```
1. 产品概述
   1.1 产品定位
   1.2 目标用户
   1.3 核心价值

2. 功能需求
   2.1 功能清单
   2.2 功能详细说明
   2.3 优先级

3. 非功能需求
   3.1 性能要求
   3.2 兼容性要求
   3.3 安全要求

4. 用户体验
   4.1 交互流程
   4.2 界面原型
   4.3 视觉风格

5. 验收标准
   5.1 功能验收
   5.2 性能验收
   5.3 用户体验验收
```
