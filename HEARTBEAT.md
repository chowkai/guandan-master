# 任务状态检查（每次 Heartbeat 必做）
- [ ] 读取 `tasks/active-tasks.json`
- [ ] 检查是否有任务到期/超时/完成
- [ ] 有变化则立即汇报给用户
- [ ] 更新任务实际耗时到基准文档

# 决策事项检查（每次 Heartbeat 必做）
- [ ] 读取 `tasks/decision-log.md`
- [ ] 检查是否有决策事项即将到期（<24 小时）
- [ ] 有到期风险则立即提醒用户
- [ ] 更新决策事项状态

# 汇报检查清单
- [ ] QQ 消息通道正常
- [ ] 记忆文件已更新
- [ ] Git 已 commit
- [ ] 文档已保存
- [ ] 无技术障碍

# 进度汇报（新增）
- [ ] 读取 `tasks/progress-tracking.md`
- [ ] 检查是否有里程碑达成（25%/50%/75%/100%）
- [ ] 检查是否有重大偏差（>50%）
- [ ] 有进展则主动汇报（不等待询问）
- [ ] 计算累计偏差和预计提前/延迟时间

# 主动汇报触发条件
1. 任务完成 → 立即汇报
2. 任务超时 → 立即汇报 + 说明原因
3. 遇到阻塞 → 立即汇报 + 请求决策
4. 提前完成 → 立即汇报
5. 到期前 1 小时 → 提醒用户
