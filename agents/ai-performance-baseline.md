## 📊 任务类型与耗时基准

### 基础任务

| 任务类型 | 平均耗时 | 波动范围 | 说明 |
|----------|----------|----------|------|
| 简单问答 | 5-15 秒 | - | 单次对话响应 |
| 复杂分析 | 30-90 秒 | - | 多步骤推理 |
| 文件读取 (小) | 5-20 秒 | - | <100KB |
| 文件读取 (大) | 30-120 秒 | - | >1MB |
| 文件写入 | 5-30 秒 | - | 含目录创建 |
| Shell 命令 | 5-60 秒 | - | 取决于命令 |

## 🧮 评估公式

```
总耗时 = 基准时间 × 复杂度系数 + Buffer

复杂度系数:
- 简单 (1x): 常规任务，无特殊要求
- 中等 (2x): 需要多步骤/多文件/跨模块
- 复杂 (5x): 需要创新/研究/多 Agent 协作

Buffer: 始终预留 50%（模型响应波动/重试/意外情况）
```

### 示例计算

**任务**: 生成 300 行代码的游戏逻辑模块

```
基准时间: 2-5 分钟 (代码生成 500 行+)
复杂度: 中等 (2x) - 游戏逻辑较复杂
计算: 3 分钟 × 2 = 6 分钟
Buffer: 6 分钟 × 1.5 = 9 分钟
评估: 约 10 分钟完成
```

---

## ⚠️ 注意事项

1. **并发限制**: 同一时间最多 8 个子 Agent 并发
2. **上下文长度**: 长上下文会增加响应时间
3. **模型波动**: 高峰期响应可能变慢
4. **网络依赖**: 外部 API/网页爬取受网络影响
5. **文件 IO**: 大文件读写较慢
